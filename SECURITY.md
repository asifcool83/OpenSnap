# Security Policy

## Supported Versions

OpenSnap has not yet published a stable release. Until v1.0, security reports should target the latest commit on `main`.

## Reporting A Vulnerability

Please do not publicly disclose security-sensitive issues before the project owner has had a reasonable opportunity to respond.

Use GitHub's private vulnerability reporting tools when available. If private reporting is unavailable, contact the repository owner through GitHub and provide:

- affected version or commit
- reproduction steps
- expected impact
- any suggested mitigation

OpenSnap does not collect telemetry, analytics, accounts, or cloud data. Security concerns are most likely to involve local macOS permissions, Accessibility behavior, build artifacts, or release distribution.
