# Motion

Motion in OpenSnap should be short, purposeful, optional, and never distracting.

## Principles

- Use motion to clarify cause and effect.
- Prefer native macOS animation timings.
- Keep animations fast.
- Never delay the user's work for decorative motion.
- Respect Reduce Motion.

## Window Movement

Window movement should prioritize speed and precision. If animation is added later, it should be subtle and configurable.

## Diagnostics And Settings

Diagnostics and settings should avoid decorative animation. State changes should be readable and calm.
