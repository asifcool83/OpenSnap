# UI

OpenSnap UI should feel like a quiet native macOS utility.

## Panels

Panels should be focused and sparse. Avoid dashboard-like layouts unless the user is explicitly debugging.

## Settings

Settings should use native macOS controls and predictable grouping. Avoid custom controls when a native control communicates the same behavior.

## Buttons

Use standard button styles. Reserve prominent buttons for the next obvious action.

## Toolbars

Toolbars should be minimal. Prefer SF Symbols with tooltips for repeated actions.

## Popovers

Popovers should be short-lived and contextual. They should not become hidden settings pages.

## Menus

Menus should be direct and keyboard-friendly. Keep labels short and action-oriented.

## Native Controls

Native controls are the default. Custom UI requires a clear reason: better clarity, better accessibility, or missing native behavior.

## macOS Conventions

Follow macOS expectations for menu bars, settings, keyboard shortcuts, permissions, focus, and accessibility prompts.
