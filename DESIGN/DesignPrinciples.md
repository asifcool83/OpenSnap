# Design Principles

These rules are permanent unless explicitly changed by project decision.

## Native Before Custom

Use macOS conventions, system controls, system colors, and SF Symbols before inventing custom UI.

## Simplicity Before Features

Do not add features simply because other window managers have them. Add features only when they make the core experience clearer, faster, or more reliable.

## Reliability Before Cleverness

Predictable behavior matters more than novel behavior.

## Performance Before Decoration

OpenSnap should feel instant. Decoration must never make the app slower or harder to understand.

## Privacy Before Data

No telemetry. No analytics. No accounts. No subscriptions. No cloud dependency for core behavior.

## Consistency Before Customization

Customization is useful only when the default experience remains coherent.

## Balance, Focus, Precision

The 60/40 identity is the product's visual and interaction anchor: one primary area for focus, one supporting area for context.
